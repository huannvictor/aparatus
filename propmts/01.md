# Regras
- SEMPRE use shadcn/ui como biblioteca de componentes.
- ANTES de criar qualquer componente, verifique se há algum do shadcn/ui disponível que atinga seu objetivo, instale-o se não estiver já instalado.
- USE APENAS as cores do tema que estão em @app\globals.css. NUNCA use cores hard-coded do Tailwind ou cores fora do tema.
- SEMPRE use os componentes que estão em @components/ui/page.tsx.
- SEMPRE use o MCP do Context7 para buscar documentações, sites e APIs.
- SEMPRE uso o componente Image do Next.js importando: `import Image from "next/image"` para renderizar imagens.
- SEMPRE use o componente Link do Next.js importando: `import Link from "next/link"` para renderizar links.
- SEMPRE crie uma função em @data ao chamar o Prisma, como é feito em @app/page.tsx:

## Requisitos Funcionais
- o botão "reservar" não deve fazer nada.
- o botão "Copiar telefone" deve copiar o telefone para o clipboard.
- os dados da barbearia devem ser recuperados do banco de dados usando o ID recebido na URL.
- a imagem do banner da página no topo deve ser a imagem da barbearia no banco de dados (imageUrl).
- o botão de "voltar" no banner da página deve retornar à pagina anterior.
- crie um componente em @components chamado 'service-item.tsx', prepare o componente para receber o serviço com prop.
- para cada serviço da barbearia recuperado no banco de dados, renderize o componente @components/service-item.tsx'. Receba o serviço como prop.
